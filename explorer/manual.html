01 — Introduction

Welcome to Chatastrophe

A local LLM playground with six purpose-built modes, four AI providers, and an unhealthy obsession with making models compete against each other.

Chatastrophe started life as a sampling parameter explorer and evolved into something considerably more interesting. At its core it's a browser-based application that runs entirely on your machine, storing all conversations and personas in a local SQLite database. Nothing leaves your environment unless you're making API calls to the cloud providers you've configured.

The app is organized into six tabs, each designed for a different kind of experiment:

🗨️ Chat 📊 Parameter Sweep 🧠 Reasoning Models ⚔️ Arena 🪑 Roundtable 🏆 Throwdown

Whether you're a researcher running systematic experiments, a developer benchmarking model choices, or just someone who wants to watch Sherlock Holmes and Hamlet debate the trolley problem — you're in the right place.

💡

Before you start: Make sure the backend server is running (uvicorn main:app --reload from the backend/ directory) and the frontend dev server is active (npm run dev from frontend/). The app runs at http://localhost:5173 by default.

03 — Getting Started

Your First Conversation

From cold start to first response in four steps.

1
Pick a provider and model

In the left sidebar, use the provider dropdown (defaults to Groq (free cloud)) and model dropdown below it. Groq requires an API key in your .env file but it's free and fast — a good starting point.

2
Create a new conversation

Click + New Chat in the sidebar, or just start typing in the input bar — a conversation will be created automatically on first send.

3
Type a message and send

Press Enter to send. Use Shift+Enter for a newline. The response streams in token by token. You can click Stop mid-stream if it's going off the rails.

4
Experiment with parameters

Slide the Temperature up in the right panel and ask the same question again. Notice how the response changes? That's the whole game. Welcome to Chatastrophe.

🎲

Not sure what to ask? Click ✨ Question ideas above the input bar. The question bank has 309 curated prompts across 8 categories, specifically chosen to produce interesting, opinionated AI responses. "Would you rather fight one horse-sized duck or a hundred duck-sized horses?" is in there. So is "What ethical framework is most compatible with utilitarianism?" You're covered.

04 — Core UI

The Conversation Sidebar

Your conversation history, model controls, and persona setup — all in the left panel.

Resizing the Sidebar

Drag the right edge of the sidebar to resize it. The width clamps between 160px (minimal) and 400px (full). Your preferred width persists while the app is open.

Provider & Model Selection

The provider dropdown lists all configured providers. Switching provider resets the model dropdown to that provider's first available model. The selected provider and model are stored with each conversation — so loading an old conversation restores the original settings.

ℹ️

Provider and model changes made in the sidebar apply to the currently active conversation and all future messages in it. They don't retroactively change the model used for earlier messages.

System Prompt

The system prompt textarea below the model selector sets the AI's behavior and persona for the entire conversation. This is where you define who the model is, how it speaks, and what constraints it operates under. You can type directly, or use the persona picker below it to auto-fill from your saved personas.

Persona Picker

The labeled dropdown below the system prompt textarea lists all saved personas. Selecting one fills the system prompt automatically. The dropdown resets after selection (it's a "pick one" control, not a persistent selection) so you can change your mind without manually clearing things.

When a conversation uses a recognized persona's system prompt, the persona name appears in indigo below the model name in the conversation list — handy for quickly scanning which character was in use.

Conversation List

Conversations are listed in the order they were created. Each entry shows:

  • The conversation title (auto-generated from the first message)
  • The model and provider in small gray text
  • The persona name in indigo, if the system prompt matches a saved persona

Click a conversation to load it. The active conversation is highlighted. Hover to reveal a delete (trash) button. Deleted conversations are gone permanently — there's no undo.

Exporting Conversations

With a conversation active, the bottom of the sidebar shows export buttons:

  • JSON — the full conversation as a structured JSON file, including all messages, parameters, and metadata
  • PDF — a formatted PDF of the conversation, suitable for sharing or archiving

These exports cover Chat conversations. Roundtable conversations are saved the same way and are also exportable.

05 — Core UI

The Parameter Panel

The right panel puts every sampling knob within reach — and tells you what each one actually does.

Toggle the panel open or closed with the ⚙️ gear icon in the top-right of the header bar. When open, the panel is 288px wide and sits to the right of the tab content.

How Parameters Work

Parameters changed in this panel take effect on the next message you send. They are also saved to the active conversation with a short debounce — so if you load the conversation later, your parameters are restored. Parameters are per-conversation, not global.

Provider-Aware Controls

Not every provider supports every parameter. Controls that aren't available for the current provider are shown grayed out with a note. Specifically:

  • Top K — Anthropic only
  • Frequency & Presence Penalty — OpenAI and Groq only
  • Fix Seed — OpenAI only
  • Token Heatmap — OpenAI only (enabled per-message in the input bar)

Tooltips

Every parameter has an ⓘ info icon next to its label. Hover it to see a plain-English explanation of what the parameter does — no documentation required. The tooltips use portal rendering, so they float above any overflow-clipping containers.

Temperature vs. Top P

These two parameters interact: if Top P is below 1.0, Temperature is ignored. The panel hints at this — when Top P is active, the Temperature slider shows "Ignored — Top P active" below it, and Top P shows "✓ Active — temperature ignored". Use one or the other, not both.

Parameter Reference

The Parameter Reference button at the bottom of the panel opens a modal with a comprehensive reference for all parameters. See also Section 14 in this manual for the full reference table.

Chat Tab

The classic chat interface, with nothing held back. Every parameter is live, every conversation is saved, and the question bank is one click away.

The Input Bar

The input bar at the bottom of the Chat tab has more going on than it appears:

Question Ideas

Click ✨ Question ideas to open a scrollable dropdown listing all 309 questions from the question bank, organized by category. Click any question to paste it into the input. The ⚙ gear icon next to it opens the Question Bank manager to add or remove questions.

Runs

The ⚡ Runs selector (values 1–5) lets you send the same message multiple times in one click. Each run generates a separate assistant response in the conversation thread. This is useful for seeing how much the model's output varies with a given temperature setting — run the same question 3 times at Temperature 1.5 and marvel at the chaos.

Token Heatmap

When using OpenAI (paid), a Token heatmap checkbox appears. Enabling it fetches log probabilities alongside the response and colors each token by confidence — green for high confidence, red for low. Note that this disables streaming for that message. The heatmap shows which words the model was uncertain about.

Message Display

Responses render as formatted Markdown — headers, bullet lists, code blocks, bold/italic text all display correctly. Code blocks get monospace font. Long responses are fully scrollable.

If you sent multiple runs, the responses appear in sequence. Each response is independent — the model doesn't see the previous runs when generating new ones in a multi-run batch.

Stopping Generation

While a response is streaming, the send button becomes a red Stop button. Click it to abort mid-stream. The partial response is discarded.

💡

Pro move: Set Temperature to 0 and Fix Seed (OpenAI only), then ask the same question twice. You should get identical responses — useful for verifying that your parameter changes are actually causing output differences and not just natural variation.

07 — Tabs

Parameter Sweep Tab

Run one prompt across a full range of parameter values and see every response side by side. Science.

The Parameter Sweep tab is the most explicitly research-oriented feature in Chatastrophe. Instead of manually adjusting a slider and re-sending, you define a parameter, a value range, and a prompt — and the app runs all the variations automatically, displaying the results in a scrollable comparison grid.

Setup

1
Choose a provider and model

Selected at the top of the Sweep view. Independent of the Chat tab's provider/model selection.

2
Write a system prompt (optional)

Sets the model's persona for all runs in the sweep.

3
Write a user message

This is the prompt that every sweep variation will respond to. Choose something where you expect the parameter to matter — creative writing, opinion questions, and brainstorming all show dramatic differences across temperature values.

4
Select the parameter to sweep and define the range

Choose from Temperature, Top P, Max Tokens, Frequency Penalty, or Presence Penalty. Set a minimum, maximum, and number of steps. The app distributes values evenly across the range.

5
Run the sweep

Results appear as a scrollable grid of cards. Each card shows the parameter value and the model's response. The sweep runs synchronously (one at a time) to avoid rate limit issues.

💡

Best sweep prompts: "Write one sentence describing the color blue" is a classic — at low temperature it's literal, at high temperature it gets genuinely weird. "Give me three creative product names for a sleep aid" is another good one. Anything where "right" answers don't exist shows the most dramatic variation.

08 — Tabs

Reasoning Models Tab

Extended thinking models deserve their own interface. Watch the chain-of-thought unfold in real time.

Some models — OpenAI's o-series and Anthropic's extended thinking Claude models — don't just answer. They think first, at length, before producing an output. The Reasoning Models tab is purpose-built for these models, exposing their internal reasoning process in a collapsible trace alongside the final answer.

Supported Models

OpenAI

o1, o1-mini, o3, o3-mini, o4-mini, and the gpt-5 family (when available). These use the reasoning_effort parameter.

Anthropic

Claude Sonnet 4.6, Claude Opus 4.6, Claude Sonnet 4.5, Claude Haiku 4.5, and related models. These use extended thinking with a configurable budget_tokens limit.

Controls

Reasoning Effort (OpenAI)

A three-position slider: low, medium, high. Higher effort means the model thinks longer before answering — more accurate on hard problems, slower, more expensive.

Budget Tokens (Anthropic)

Sets the maximum number of tokens the model can spend on internal reasoning before producing its response. Higher budget = more thorough thinking = higher cost. Start with 5,000 for everyday use; bump to 16,000+ for genuinely hard problems.

Max Tokens

The maximum length of the output (not counting the reasoning tokens). The reasoning trace doesn't count against this limit.

The Reasoning Trace

During streaming, the reasoning trace appears first — you can watch the model work through the problem step by step. Once reasoning is complete, the final answer streams out. After completion, the reasoning trace is collapsible (click to toggle). The trace is preserved with the conversation so you can review it later.

🧠

Try this: Ask o3 and Claude Opus to solve the same logic puzzle. Then compare their reasoning traces. They approach problems very differently — one tends to enumerate cases, the other tends to reason by elimination. Genuinely fascinating if you're into that sort of thing.

⚠️

Cost note: Extended thinking models can be significantly more expensive per query than standard models. A single query with a 16,000 budget_tokens limit on Claude Opus can cost several cents. Keep this in mind if you're experimenting at scale.

Setup

Each column has its own provider and model selector at the top. The left and right columns are fully independent — you can compare any two models across any two providers. Mix OpenAI with Anthropic, Groq with Ollama, or run the same model twice with different parameter settings.

Roundtable Tab

Put two to four AI personas in a room together and give them a topic. Watch a conversation happen. Steer it wherever you want.

Overview

Roundtable runs a multi-agent conversation where each agent has its own persona, system prompt, provider, and model. A moderator picks speakers using a recency-based weighting algorithm — agents who just spoke are weighted down so conversations don't get stuck in a two-way loop. The result feels like a genuine (if occasionally unhinged) discussion.

Adding and Configuring Agents

Roundtable starts with two agents. Click + Add agent to add a third or fourth (maximum four). Each agent card contains:

  • A persona picker dropdown to select from saved personas
  • An editable system prompt textarea (auto-filled when a persona is selected)
  • A provider and model selector — each agent can use a completely different model
  • A remove button (×) that appears when there are more than two agents

Topic and Turns

Enter a topic or conversation starter in the topic field. The Turns slider (3–16) sets how many total speaking turns the conversation runs. With four agents and 12 turns, each agent speaks roughly 3 times.

Tone Sliders

Two sliders shape the feel of the entire conversation by injecting tone instructions into every agent's prompt at runtime:

  • Mood (😔 Gloomy → 😄 Cheerful) — affects emotional register across all agents
  • Seriousness (🎭 Playful → 🎓 Serious) — affects how formally and earnestly agents engage

These are "prompt injection" sliders — they add language like "respond in a cheerful tone" or "be playful and witty" to each agent's system prompt. No magic, just text.

Follow-Up Topics

After a conversation completes, a follow-up input appears. Enter a new topic or question and run another round — the agents continue the same conversation thread with the new direction. Great for steering a discussion or probing a specific angle that came up.

Saving and Resuming

Roundtable conversations are saved to the database with the special system prompt marker __roundtable__. They appear in the conversation sidebar. When you click a roundtable conversation in the sidebar, it automatically switches to the Roundtable tab and restores the conversation history so you can continue from where you left off.

🪑

Best configuration: Set Mood to maximum Cheerful, Seriousness to maximum Playful, and have Nietzsche, Sherlock Holmes, and Jane Austen debate whether hot dogs are sandwiches. You're welcome.

11 — Tabs

Throwdown Tab

Four personas enter. One leaves a champion. The AI game show you didn't know you needed.

Concept

Throwdown is a competitive format: four personas compete over a user-configured number of rounds. One persona is designated the Judge. Each round, all three Contestants respond to the same prompt, and the Judge reviews all three responses and picks a winner — explaining their reasoning in 2–3 sentences. The contestant with the most wins at the end is declared champion.

The Judge is fixed for the entire session. This is intentional — a rotating judge would introduce a conflict of interest (whoever's behind might favor other contestants when it's their turn to judge). Fixed judge = clean data.

Auto-Queued Questions

When you click Start Throwdown, the app randomly selects N questions from the question bank (where N is your round count) and queues them up. The first question is pre-filled in the prompt input. After each round completes, the next question automatically loads. You can edit the pre-loaded question before running the round, or use the ✨ sparkle button to browse the full bank for alternatives.

Running a Round

Click ▶ Run (or press Enter) to start a round. The three contestants respond sequentially — you watch each one stream in, one at a time. After all three are done, the Judge's verdict streams in automatically. The round is scored and the next prompt loads.

The Scoreboard

The sticky header shows the current round number, each contestant's score, and the Judge's name. Scores update immediately after each verdict. Because scoring is keyed by contestant slot (not by persona name), you can put the same persona with different models as contestants and get correct, separate scores for each.

Same Persona, Different Models

This is a key use case: run "GPT-4o as Sherlock Holmes" against "Claude Sonnet as Sherlock Holmes" against "Llama 70b as Sherlock Holmes" and have a fourth persona judge them. When two contestants share a persona name, their column headers automatically show the model name in parentheses to disambiguate — "Sherlock Holmes (gpt-4o)" vs "Sherlock Holmes (claude-sonnet)". The judge prompt uses neutral labels (Contestant A/B/C) to avoid any name-based bias.

Phase 3: Results

After all rounds complete, the Complete screen shows:

  • The champion with their win count
  • A final scoreboard sorted by wins
  • A collapsible Round History showing every round's responses and judge verdicts
  • A New Session button to start over
🏆

Best experiment: Put the same persona (say, "a snarky assistant") on GPT-4o, Claude Sonnet, and Llama 3.3 70b, with Oprah Winfrey as judge. Run 7 rounds of Hot Takes questions. The results are eye-opening — and occasionally horrifying.

12 — Content

Personas

Saved persona configurations — the characters that populate your Roundtables and Throwdowns.

A persona is a named character with an associated system prompt. Personas are stored in the SQLite database and are available everywhere in the app — the Chat sidebar's persona picker, the Roundtable agent setup, and the Throwdown agent cards all pull from the same list.

The Persona Manager

Access the Persona Manager via the Personas button in the conversation sidebar (near the bottom of the sidebar controls). The modal shows all saved personas in a scrollable list.

Creating a Persona

Click + New Persona in the manager. Fill in:

  • Name — the persona's display name (e.g., "Nietzsche", "Sherlock Holmes")
  • System Prompt — the full system prompt that defines how this character speaks and behaves. This is the most important field — a well-crafted system prompt makes a huge difference.
  • Book / Author / Show — optional metadata fields for organizing your library

Editing a Persona

Click the edit (pencil) icon on any persona row to modify it. Changes are saved immediately.

Deleting a Persona

Click the delete (trash) icon. Permanent, no confirmation. Deleted personas don't affect existing conversations that used them — the system prompt text is stored with each conversation, so old conversations remain intact.

Writing Good System Prompts

The quality of your persona experience is almost entirely determined by the system prompt. Some guidelines:

  • Be specific about voice. "Respond as Nietzsche" is weak. "You are Friedrich Nietzsche. Speak with intellectual intensity. Use aphorisms freely. Challenge the conventional moral frameworks of whoever is speaking. You have contempt for weakness and pity." is strong.
  • Define what the character knows and doesn't know. A Victorian-era character shouldn't reference smartphones unless that tension is the point.
  • Give the character opinions. Generic characters produce generic responses. Give them pet peeves, strong stances, and recurring themes.
  • Set the register. Formal? Casual? Theatrical? The model will follow your lead.
💡

Test before committing: Create a persona and immediately test it in the Chat tab with a few questions before using it in a Throwdown. You want to make sure the character is doing what you expect before you run a 10-round session with it.

14 — Reference

Sampling Parameters Reference

What every slider actually does, in plain English — no probability theory required.

Parameter Range Support What it does
temperature 0 – 2 (0–1 for Anthropic) All Controls creativity and randomness. At 0, the model always picks the most likely next token — deterministic, focused, repetitive. At 2, it picks more surprisingly — creative, unpredictable, occasionally incoherent. 0.7 is a good all-purpose starting point. Ignored if Top P is below 1.0.
top_p 0 – 1 All Nucleus sampling. Restricts the model to only consider tokens that together make up the top P% of probability mass. At 1.0 it has no effect. Below 1.0, it overrides Temperature. Use one or the other — not both.
top_k 0 – 500 Anthropic Restricts sampling to the top K most likely next tokens. At 0, disabled. Lower values = more focused responses, less variety. Higher values = more varied word choices. Works in combination with Temperature.
max_tokens 1 – 4096 All Hard cap on response length in tokens (~¾ of a word each). The model stops generating once it hits this limit, even mid-sentence. If responses are getting cut off, raise this. If you want brief answers, lower it.
frequency_penalty -2 – 2 OpenAI Penalizes tokens proportionally to how many times they've already appeared. Positive values discourage repetition of specific words. Negative values actively encourage repetition. Above 1.5, responses can become stilted as the model avoids even normal repeated words.
presence_penalty -2 – 2 OpenAI A flat penalty for any token that has appeared at all in the response so far. Unlike frequency_penalty, it doesn't scale with frequency — just "has this appeared or not". Positive values encourage topic-hopping. Negative values encourage staying on topic.
seed Any integer OpenAI only When enabled (Fix Seed checkbox), the model uses this seed value to make sampling deterministic. Same prompt + same seed = same response. Invaluable for comparing parameter changes without noise from random variation.
stop_sequences Comma-separated strings All The model immediately stops generating when it outputs one of these exact strings. Useful for controlling response format. Example: enter END and tell the model "finish your response with the word END" to guarantee clean termination.
ℹ️

Groq supports Temperature, Top P, Max Tokens, Frequency Penalty, and Presence Penalty. It does not support Top K, Seed, or Token Heatmap. Ollama support depends on which models you have loaded locally.

15 — Reference

Providers & Models

Four providers. Configure your API keys and you're in.

Groq (free cloud)

The default provider. Groq runs open-source models on their LPU hardware — extremely fast inference with a generous free tier. Best for casual use, rapid experimentation, and Throwdown sessions where you want snappy responses. Requires a free Groq API key.

Featured models: llama-3.3-70b-versatile, llama-3.1-8b-instant, gemma2-9b-it, mixtral-8x7b-32768

Ollama (local)

Run models entirely on your own machine — no API key, no cost, no data leaving your computer. Requires Ollama to be installed and running (ollama serve). Speed depends on your hardware. Models must be pulled with ollama pull <model-name> before they appear as available.

Best for: privacy-sensitive use cases, offline use, experimenting with open-source fine-tunes.

OpenAI (paid)

Access to GPT-4o, GPT-4o-mini, and the o-series reasoning models. Requires an OpenAI API key with billing configured. This provider enables features not available elsewhere: Token Heatmap (log probabilities), Seed fixing, and the Reasoning Models tab for o1/o3/o4-mini.

Anthropic (paid)

Access to the Claude family — Claude Sonnet 4.6, Claude Opus 4.6, and others. Requires an Anthropic API key. This provider enables Top K sampling and extended thinking via the Reasoning Models tab.

ℹ️

API keys are configured in a .env file in the backend/ directory. The backend reads GROQ_API_KEY, OPENAI_API_KEY, ANTHROPIC_API_KEY. Providers without a configured key will return errors when you try to use them.

16 — Advanced

Power User Tips

Things that aren't obvious until you've been using the app for a while.

Keyboard Shortcuts

KeyAction
EnterSend message (Chat, Throwdown, Roundtable input bars)
Shift+EnterNew line in message input

Parameter Experiments

  • Isolate variables. If you're comparing Temperature effects, fix everything else. Fix Seed (OpenAI) makes this rigorous.
  • Use the Sweep tab for range testing rather than manually sending the same message 10 times at different temperatures. That's what it's for.
  • Temperature 0 + Max Tokens 50 = concise, deterministic answers. Good for structured extraction tasks.
  • Frequency Penalty 1.0 + creative writing prompts produces noticeably more varied vocabulary. Worth experimenting with.

Roundtable Strategies

  • Don't give agents too much common ground. Interesting conversations happen at the friction points between worldviews. Put an optimist and a pessimist in the same roundtable.
  • Start with a strong, specific topic. "AI" produces generic responses. "Should AI systems be granted legal personhood, and if so, what rights should they have?" produces actual debate.
  • Use Mood slider at extremes. Maximum Cheerful + Maximum Playful makes philosophical debates feel like game shows. Maximum Gloomy + Maximum Serious turns hot dog taxonomy into existential crisis. Both are fun.
  • 16 turns with 4 agents gives each agent 4 turns — long enough for opinions to develop and for agents to start responding to what others said in earlier turns.

Throwdown Strategies

  • Judge character matters. A Judge with a strong personality produces more interesting verdicts. "A ruthless Wall Street trader" as judge produces different decisions than "a compassionate kindergarten teacher." Both are legitimate experimental setups.
  • Hot Takes + mismatched personas = chaos. "Should pineapple go on pizza?" asked of Nietzsche, Shakespeare, and a medieval knight produces responses that are inexplicably revealing.
  • Use Would You Rather for clean, decisive judging. The binary format makes it easy for the judge to pick a winner based on reasoning quality rather than response length.
  • Same persona, different models is genuinely valuable data. If you're deciding which model to use for a character-based application, Throwdown gives you side-by-side comparative data with a consistent scoring methodology.

Data & Storage

  • All data lives in backend/explorer.db (SQLite). You can back it up by copying the file.
  • Conversations, personas, questions, Throwdown sessions, and Throwdown rounds are all in the same database.
  • The JSON export from the sidebar is the cleanest format for sharing individual conversations. The PDF export is better for reading.
🔥

The ultimate Chatastrophe session: Start a Roundtable between 4 historically incompatible thinkers on a topic they'd all have strong opinions about. Set Mood to maximum Cheerful (so they're weirdly upbeat about their disagreements). Let it run 16 turns. Save the conversation. Export as PDF. Read it the next morning. It's better than you remember.
